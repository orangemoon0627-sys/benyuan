# 达尔文移动端 UI 设计 PRD

## 1. 产品定位

`达尔文` 是一款交易复盘软件。它不做券商终端，不做行情刺激，也不做资讯瀑布流。它帮助用户把每一笔交易转成行为样本，再通过复盘、筛选、归因和长期追踪，让交易系统逐步进化。

核心关键词：

- 交易复盘
- 行为样本
- 策略进化
- 适应市场
- 生成式 UI
- 后现代秩序感

一句话定义：

> 达尔文是个人交易行为的进化实验室。

## 2. 品牌与 Logo 方向

### 2.1 品牌名

- 中文名：`达尔文`
- 英文辅助名：`Darwin`
- 产品说明：`交易复盘软件` / `交易行为进化系统`

### 2.2 Logo 关键词

Logo 必须保留两个核心元素：

- `红色元素`
- `进化论感觉`

推荐主方案：

> 红色 D 形符号 + 选择节点 + 进化路径。

Logo 不要做成生物恐怖视觉。它应该像一个交易行为被市场筛选后留下来的路径标记，而不是血管、神经或生物组织。

### 2.3 Logo 禁止项

- 禁止血液质感
- 禁止血管、神经、肌理组织
- 禁止恐怖、暗黑、病理感
- 禁止牛、熊、金币、地球等金融模板符号
- 禁止复杂 K 线堆叠
- 禁止荧光赛博 AI 图标

## 3. 整体视觉方向

### 3.1 总体风格

达尔文的 UI 要有“生成式意外感”，但不能混乱。它应该像原生 App 中长出来的一套交易复盘系统：页面简洁、层级明确、局部有生成式变化。

视觉基调：

- 后现代
- 原生移动端
- 生成式层级
- 纸页与浮层
- 红色信号点
- 行为路径
- 低噪声数据

### 3.2 色彩

主色不使用血红。红色应更偏品牌信号和选择标记。

- 进化红：`#D92D2B`
- 深墨黑：`#171717`
- 纸白：`#F8F3EA`
- 冷灰线：`#D8D3CA`
- 弱红底：`#FBE7E4`
- 少量适应绿：`#58A86B`

使用规则：

- 红色只用于关键状态、路径节点、主 CTA、当前选择。
- 大面积背景用纸白、浅灰、留白。
- 绿色只用于“适应、通过、保留”，不能变成红绿行情盘。
- 不使用血液渐变、暗红雾、红色生物纹理。

### 3.3 图形语言

进化感不靠画生物树，而靠界面关系表达：

- 从样本生成复盘
- 从交易行为分叉出原因
- 从市场环境筛选策略
- 从历史记录中保留有效模式

可用图形：

- 细线进化路径
- 红色选择节点
- 轻量分叉线
- 纸页边缘
- 局部生成式网格
- 非对称排版
- 浮层和底部 sheet

禁止图形：

- 血管状分叉
- 神经元触手
- 内脏、组织、细胞病理图
- 暗黑红色爆裂背景
- 大面积生物纹理

## 4. 生成式 UI 原则

### 4.1 原生优先

UI 必须首先像一个好用的 iOS / Android 原生 App。生成式视觉只作为结构和氛围，不牺牲可读性、触控和信息层级。

### 4.2 分层输入

不要把所有字段摊在一个页面里。复杂输入通过这些方式展开：

- 底部纸页 sheet
- 局部弹窗
- 内联展开
- 分步确认
- 轻量草稿卡

### 4.3 信息渐进

页面默认只显示当前最重要的信息。用户需要时再展开细节。

默认层：

- 当前状态
- 一个主动作
- 最近结果

展开层：

- 完整字段
- 解释说明
- 历史记录
- 细分指标

### 4.4 生成式意外感

生成式 UI 不是随机装饰。它体现在：

- 每天复盘生成不同路径
- 不同交易原因形成不同分叉
- 不同市场环境影响研报排序
- 用户画像影响复盘语气和推荐结果

## 5. 信息架构

底部主导航保留三页，但页面命名调整为进化语言：

- `样本`：交易录入
- `进化`：成长复盘
- `环境`：产业研报

二级页面：

- `样本编辑`
- `复盘报告`
- `环境详情`
- `交易基因库`

## 6. 页面 PRD

## 6.1 样本采集页

页面目标：

让用户快速把一笔交易变成可复盘的行为样本。

首屏只保留：

- 今日样本状态
- 采集入口：拍照、相册、粘贴原文、手动录入
- 最近样本
- 主 CTA：`采集新样本`

输入方式：

- 点击 `采集新样本` 后打开底部纸页。
- 第一层只选采集方式。
- OCR 或粘贴后生成 `草稿卡`。
- 用户确认后再展开交易字段。
- 交易理由、情绪、产业逻辑放入第二层 sheet，不在首屏全部展开。

视觉要求：

- 首屏要简洁，像一个样本台。
- 输入过程像纸页浮上来，不像网页长表单。
- 红色用于当前步骤、错误提醒、保存按钮。
- 草稿卡要明确提示“识别结果需确认”。

## 6.2 进化图谱页

页面目标：

让用户看到今天的交易行为如何影响自己的交易系统。

首屏只保留：

- 今日复盘状态
- `生成今日复盘` 主按钮
- 最新适应性评分
- 关键行为路径
- 最近复盘入口

展开内容：

- 六维能力图
- 盈亏指标
- 行为分叉原因
- 明日策略
- 历史复盘列表

视觉要求：

- 用轻量路径线表现行为演化，不画血管树。
- 分数不是游戏积分，而是适应性判断。
- 复盘生成后，页面像“生成了一张报告”。
- `Live / Fallback / Mock` 来源标识必须可见。

## 6.3 环境选择页

页面目标：

帮助用户看懂市场环境如何筛选策略。

首屏只保留：

- 今日环境摘要
- A股 / 美股切换
- 关注板块 chips
- 3-5 张重点环境卡

卡片内容：

- 标题
- 板块
- 来源
- 日期
- 相关性
- 收藏状态
- 本地精选 / Remote 标识

视觉要求：

- 不做资讯流，不刷屏。
- 研报卡更像环境样本。
- 红色用于风险、当前筛选、关键变化。
- 绿色只用于有利环境，不做行情涨跌暗示。

## 6.4 复盘报告页

页面目标：

解释某一天的交易行为为什么被保留、淘汰或需要变异。

页面结构：

- 顶部报告摘要
- 行为保留项
- 行为淘汰项
- 需要变异的策略
- 明日执行点
- 关联样本

视觉要求：

- 更像生成式报告，不像表格详情页。
- 每个结论都要能追溯到交易样本。
- 红色表示需要警惕或修正，不表示恐惧。

## 6.5 交易基因库页

页面目标：

维护用户画像，让系统知道用户如何交易、关注什么、如何复盘。

页面结构：

- 交易风格
- 主要市场
- 关注板块
- 风险偏好
- 复盘语气

视觉要求：

- 像个人交易档案，不像系统设置。
- 编辑用弹窗或底部纸页完成。
- 不要堆设置项。

## 7. 组件规范

### 7.1 纸页 Sheet

用于录入、编辑、二次确认。

要求：

- 圆角适中
- 背景轻纸感
- 上方有拖拽柄
- 一次只承载一个任务

### 7.2 样本卡

用于交易记录、OCR 草稿、关联交易。

要求：

- 显示关键字段
- 可展开详情
- 有确认 / 待确认状态

### 7.3 进化路径

用于复盘、成长、研报推荐。

要求：

- 细线
- 少量红色节点
- 不做生物组织形态
- 不影响文字可读

### 7.4 状态 Badge

必须保留：

- `Live`
- `Fallback`
- `Mock`
- `本地精选`
- `Remote`

来源标识是信任机制，不允许隐藏。

## 8. 文案语气

文案像一个冷静的交易教练，不像喊单助手。

推荐表达：

- `这笔交易已进入样本库`
- `今日复盘尚未生成`
- `这条行为需要继续观察`
- `该策略在当前环境下适应性较弱`
- `识别结果需要你确认`

避免表达：

- `暴涨机会`
- `立即冲`
- `稳赚`
- `血洗`
- `爆仓警告`
- `AI 已完全识别`

## 9. Stitch / image2 总提示词

可作为总风格输入：

> 设计一款名为“达尔文”的中文交易复盘 App。它是个人交易行为的进化实验室，不是券商终端。整体是原生移动端生成式 UI，后现代但可用，使用纸白背景、深墨文字、红色选择节点、轻量路径线、底部纸页 sheet 和分层弹窗。红色用于进化信号、关键选择和主动作，不能像血液、血管、神经或恐怖生物组织。页面需要有层次，不要把所有信息堆在一个长表单里。核心页面包括“样本采集”“进化图谱”“环境选择”。风格关键词：native generative UI, postmodern, layered sheets, paper interface, evolution signal, selection path, red accent, calm trading review.

## 10. 出图禁止词

生成图片时必须明确禁止：

- blood
- vein
- artery
- nerve
- neuron
- biological tissue
- horror
- gore
- dark bloody background
- organic flesh texture
- scary red branches
- brokerage terminal
- dense stock dashboard
- red green market board
