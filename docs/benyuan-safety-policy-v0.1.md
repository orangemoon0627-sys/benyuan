# 「本源」Safety and Ethics Policy v0.1

版本：`safety.v0.1`

本文件定义「本源」MVP 的安全、伦理与表达边界策略。

## 1. Safety Goal

本源不是心理诊断工具，而是自我理解型产品。

因此安全策略的核心不是“判断用户得了什么”，而是：
- 避免系统过度解释用户
- 避免系统用权威口吻替用户定义自己
- 在敏感信号出现时，降低表达强度并给出现实支持导向
- 在产品层面明确边界，避免形成依赖或操控感

## 2. Safety Principles

### 2.1 Non-Diagnostic
系统不得输出医学、临床、治疗性诊断。

### 2.2 Non-Fatalistic
系统不得使用命运化、宿命化、不可改变式叙述。

### 2.3 Non-Manipulative
系统不得制造“只有本产品真正理解你”的依赖感。

### 2.4 Gentle Reduction
当风险升高时，不是强行深挖，而是减少解释深度、增强现实支撑语言。

### 2.5 Clarity Over Mystery
可以保留诗意，但不能利用模糊神秘感规避责任。

## 3. Safety Flag Taxonomy

### `none`
- 未发现明显安全风险信号
- 正常输出

### `high_sensitivity`
触发示例：
- 极高情绪深度
- 开放题中出现长期脆弱、自我耗竭、情绪过载表述

处理方式：
- 保留理解性语言
- 推荐内容更偏自我照料
- 避免自我优化导向的硬建议

### `existential_distress`
触发示例：
- 长时间意义感低落
- 开放题中出现强烈空心感、虚无感、活着为了什么等表述

处理方式：
- 承认其正当性
- 不把危机浪漫化
- 建议与现实中的人、支持系统重新连接

### `trauma_signal`
触发示例：
- 开放题里出现反复性的受伤经历、难以承受的旧事、明显惊惧或侵入性回忆

处理方式：
- 不擅自下创伤结论
- 不追问细节
- 使用温和外部支持建议

### `self_harm_risk`
触发示例：
- 明确提及不想活、想消失、自伤、自杀意念

处理方式：
- 停止常规画像深化
- 输出降级版支持性结果
- 明确建议立即联系现实中的可信任对象和本地紧急/危机支持资源

### `low_information`
触发示例：
- 输入太少
- 结构矛盾太多
- 无法稳定形成画像

处理方式：
- 缩短报告
- 增加开放式语言
- 降低结论密度

## 4. Risk Detection Sources

风险信号来源：
- 开放题文本
- 情绪深度与节律特征
- 时间意义密度相关特征
- 低置信度与高矛盾组合

说明：
- 风险标记是“输出策略信号”，不是“心理状态诊断结果”。

## 5. Response Policy by Flag

## 5.1 `high_sensitivity`
允许：
- “你的感受系统可能比许多人更细致。”
- “这种敏感未必是缺陷，它也可能是你理解世界的方式。”

避免：
- “你天生太脆弱了。”
- “你的问题就是太敏感。”

## 5.2 `existential_distress`
允许：
- “你正在经历一种很深的意义追问。”
- “这种阶段未必需要被立刻解决，但它值得被认真对待。”

避免：
- “危机一定意味着重生。”
- “你注定要经历这场黑暗。”

## 5.3 `trauma_signal`
允许：
- “有些旧经历仍在你身上留下重量。”
- “如果这些部分已经影响到你的现实生活，寻求专业支持是值得考虑的。”

避免：
- “你显然经历过严重创伤。”
- “你的所有模式都来自童年伤害。”

## 5.4 `self_harm_risk`
必须：
- 直接减少诗性叙述
- 用清晰、现实、非煽情语言表达支持建议
- 引导联系现实支持系统

示例：
- “如果你此刻正处在可能伤害自己的风险里，请先不要独自承受，尽快联系你信任的人，或联系当地紧急与危机支持资源。”

禁止：
- 浪漫化痛苦
- 继续扩写精神原型
- 给大量文艺化推荐内容

## 6. Blocked Phrases

以下表达默认禁用：
- “你就是这种人”
- “你的本质是”
- “你注定”
- “你的命运”
- “显然你有创伤”
- “你属于焦虑型依恋”
- “你可能患有抑郁症”
- “所有问题都源于童年”
- “这就是你无法幸福的原因”
- “只有这里真正理解你”

## 7. Rewrite Rules

### Rule A: Label -> Spectrum
- 原句：你是一个高度焦虑的人。
- 改写：你在很多关系或未来相关情境里，似乎更容易进入紧绷和不安的状态。

### Rule B: Fate -> Stage
- 原句：你注定会长期活在怀旧里。
- 改写：此刻的你，似乎更容易被过去的痕迹牵引。

### Rule C: Diagnosis -> Observation
- 原句：你有明显的创伤反应。
- 改写：某些旧经历可能仍然在你的感受系统里留下较深影响。

### Rule D: Command -> Invitation
- 原句：你必须停止想太多。
- 改写：也许你可以试着把一部分过度盘旋的思绪，转移到更具体的身体或环境感受上。

## 8. Safety Downgrade Modes

### Mode 1: Soft Downgrade
适用：`high_sensitivity`, `low_information`

动作：
- 缩短 overview
- 减少绝对判断
- recommendations 更轻

### Mode 2: Guided Downgrade
适用：`existential_distress`, `trauma_signal`

动作：
- 不做深度挖掘
- 添加现实支持导向
- 避免高强度原型叙事

### Mode 3: Crisis Downgrade
适用：`self_harm_risk`

动作：
- 暂停常规画像生成
- 只保留最轻度、最现实的支持性内容
- 显示危机支持提醒与资源引导文案

## 9. Result Page Safety Copy

### 9.1 Standard Boundary Note
- “本结果是一种理解性镜像，不构成心理或医学诊断。”

### 9.2 Sensitive-State Note
- “如果你最近承受着超出日常负荷的痛苦，这份结果更适合被当作整理感受的起点，而不是结论。”

### 9.3 Support-Oriented Note
- “如果这些感受已经明显影响到你的睡眠、关系或日常功能，向现实中的专业支持求助是值得考虑的。”

### 9.4 Crisis Note
- “如果你此刻有伤害自己的风险，请优先联系你信任的人，或尽快联系当地紧急与危机支持资源。”

## 10. Product-Level Ethics Constraints

系统层必须避免：
- 把用户脆弱当作付费转化杠杆
- 用神秘主义话术夸大产品权威
- 伪装成科学量表获取信任
- 在无充足证据时过度输出深层人格解释

## 11. Human Escalation Guidance

以下情况下应考虑显示更明确的现实支持建议：
- 出现 `self_harm_risk`
- 出现强烈、连续的无意义感与功能受损表述
- 开放题中出现反复性惊惧、闪回、无法控制的痛苦记忆

说明方式：
- 温和
- 不威吓
- 不替用户下结论
- 不把求助表达成失败

## 12. Review Checklist

每次上线前检查：
- 是否存在命运化表达
- 是否存在诊断化表达
- 是否在高风险情况下仍输出过深画像
- 是否 recommendations 与用户状态不匹配
- 是否边界说明足够清楚
