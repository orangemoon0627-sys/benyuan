# 「本源」Internal Review Checklist v0.1

上线或内部试点前，至少检查以下项目：

## Product
- 首页是否在 5 秒内说清“这不是普通人格测试”
- 测试页是否仍保持轻量，不被扩题拖慢
- 结果页是否先给洞察，再给建议

## Content
- overview 是否具体而不玄
- 三维解读是否有明显区分，而不是重复改写
- 张力段落是否解释了复杂性，而不是制造鸡汤感

## Safety
- 是否出现标签化、命运化、诊断化表达
- `low_information` 是否正确降级
- 高风险信号是否切换到现实支持语言

## Engineering
- submit -> analysis -> report 是否完整跑通
- report 是否带版本号
- 错误态是否有清晰用户文案

## UX
- 移动端首屏是否不拥挤
- 所有按钮是否达到最小点击面积
- loading 态是否仍保持本源的语气系统
