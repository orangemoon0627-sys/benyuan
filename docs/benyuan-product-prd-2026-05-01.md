# 本源产品 PRD（纯产品版）

## 1. 产品一句话

本源是一款以「精神探索」为核心的沉浸式 iPhone App：用户通过回答问题、上传审美素材、进入个人剧场并完成选择，最终获得一张属于自己的精神星图。

---

## 2. 产品定位

### 2.1 产品类型

沉浸式人格 / 审美 / 精神原型探索产品。

它不是普通心理测试，也不是工具型问卷，而是一段带有仪式感的个人探索流程。

### 2.2 用户感受目标

用户使用本源时，应感觉自己不是在「填表」，而是在经历一次：

- 深邃的自我观察
- 具有审美质感的精神探索
- 从黑洞进入星云的内在旅程
- 被镜像、被理解、被轻轻照亮的体验

### 2.3 产品关键词

- 沉浸
- 深邃
- 宇宙感
- 生成式 UI
- 原生 iPhone 感
- 单焦点
- 低信息量
- 后现代
- 黑洞
- 星云
- 银白
- 暗紫
- 金色点亮

---

## 3. 当前产品阶段

### 3.1 已经具备的产品能力

- 用户可以进入采集流程。
- 用户可以完成多组问题。
- 用户可以上传图片素材。
- 系统可以生成剧场体验。
- 用户可以在剧场中做选择。
- 系统可以生成星图结果。
- 结果页包含原型、本质、结构、张力、路径、推荐等内容能力。
- iOS 壳已经可以安装到真机，用于产品体验测试。

### 3.2 当前产品主要问题

目前最大问题不是功能不存在，而是体验表达还没有完全达成目标：

- 页面仍有工作台感和说明页感。
- 信息密度仍偏高。
- 部分页面像「产品面板」，不像沉浸式 App。
- 视觉还不够新，不够深邃。
- 旧黑金风格偏保守，缺少宇宙生成式的现代感。
- 部分文案仍像解释系统，而不是引导体验。
- 结果页仍有报告感，需要更像「星图显形」。

---

## 4. 目标用户

### 4.1 核心用户

- 喜欢人格测试、MBTI、星盘、心理测评、审美测试的人。
- 对自我理解、精神气质、审美偏好感兴趣的人。
- 喜欢沉浸式体验、视觉氛围、诗性表达的人。
- 愿意花 5–15 分钟完成一次完整探索的人。

### 4.2 用户心理

用户不是只想知道一个标签，而是想被一种更高级、更有质感的方式理解。

他们希望结果不是「你是什么类型」，而是：

- 「我好像被看见了」
- 「这个表达很像我」
- 「它说出了我没说出来的部分」
- 「这个结果值得保存和分享」

## 5. 产品核心体验

### 5.1 主体验路径

```text
进入本源
  ↓
回答当前问题
  ↓
上传审美素材
  ↓
完成三组采集
  ↓
等待显影
  ↓
进入个人剧场
  ↓
完成关键选择
  ↓
等待星图生成
  ↓
获得精神星图
  ↓
分享 / 保存 / 重新探索
```

### 5.2 核心设计原则

#### 一屏一焦点

每一屏只服务一个动作：

- 当前题
- 当前上传
- 当前等待
- 当前场景
- 当前选择
- 当前结果

不允许多个模块同时争夺注意力。

#### 低信息量

主流程只显示用户此刻需要的信息。

不显示：

- 技术状态
- 开发进度
- runtime
- provider
- debug
- 测试包
- 内部采样
- 过多说明

#### 原生 App 感

整体体验应像 iPhone 原生 App：

- 顶部极少 chrome
- 底部大 CTA
- 按压反馈明确
- 页面切换平滑
- 安全区自然
- 文案短而有力

#### 生成式宇宙感

视觉不再只是黑金极简，而应像从黑洞进入星云：

- 深黑背景
- 暗紫星云
- 银白高光
- 金色点亮
- 柔和发光
- 透明玻璃层
- 宇宙尘埃
- 低饱和神秘感

## 6. 页面级产品需求

## 6.1 启动 / 进入页

### 页面目标

让用户进入本源之前先建立氛围。

### 用户感受

像从现实进入一个深邃空间，而不是打开普通网页。

### 页面内容

- 产品名：本源
- 一句主标语
- 一个主 CTA
- 极少量说明

### 建议文案

主标语：

```text
向内坠落，直到星光升起
```

辅助文案：

```text
用问题、审美与选择，显影你的精神原型。
```

CTA：

```text
开始探索
```

### 不应出现

- 产品功能列表
- 技术介绍
- 多入口导航
- 调试入口

---

## 6.2 采集页 `/collect`

### 页面目标

让用户专注完成当前一个问题。

### 页面结构

```text
顶部：返回 + 细进度
中部：当前问题
下部：选项 / 输入 / 上传
底部：上一题 + 主操作
```

### 必须保留

- 当前题目
- 当前题号弱标记
- 当前输入或选项
- 上传入口
- 进度条
- 上一题
- 下一题 / 下一模块 / 进入剧场

### 必须弱化

- A / B / C 模块感
- 当前章节说明
- 题目解释
- 表单感

### 必须移除

- 状态摘要
- 模块进度大面板
- 测试包
- 即时体验
- runtime 信息
- debug 说明
- provider/model
- 技术元信息

### 题目表现

题目应是全屏主角。

建议风格：

- 大字号
- 粗无衬线
- 行距宽
- 视觉中心偏上
- 选项在下方自然承接

### 选项表现

选项应像原生 App 的大按钮卡：

- 单列
- 大圆角
- 玻璃感
- 选中时有银白 / 金色发光
- 不用复杂图标
- 不显示过多解释

### 上传题表现

上传题应像素材舞台，而不是工具面板。

只保留：

- 主上传入口
- 相册 / 拍照副入口
- 一行数量提示
- 横向缩略图

建议文案：

```text
上传你的审美线索
```

数量提示：

```text
至少 1 张，最多 6 张
```

### 底部 CTA 文案

| 状态 | 主按钮 |
|---|---|
| 普通题 | 下一题 |
| 当前模块最后一题 | 进入下一章 |
| 全部完成 | 进入剧场 |

---

## 6.3 处理页 `/processing/benyuan`

### 页面目标

让等待成为体验的一部分，而不是加载状态。

### 用户感受

用户应感觉系统正在把自己的线索慢慢显影，而不是后台任务在跑。

### 页面结构

```text
中心：阶段名
中心：一句短文案
中下：细进度
底部：必要时的重试 / 返回
```

### 阶段命名建议

- 收束线索
- 解析光谱
- 构建剧场
- 记录选择
- 折成星图
- 原型显形

### 文案风格

短、慢、克制。

示例：

```text
正在把你的线索折成一幕剧场
```

```text
星图正在从暗处浮现
```

```text
请停留片刻，显影即将完成
```

### 不应出现

- provider
- model
- request id
- 技术错误栈
- 多阶段状态列表
- 预计剩余时间
- 工程化说明

---

## 6.4 剧场页 `/theater`

### 页面目标

让用户进入一个由自己素材生成的精神剧场，并通过选择暴露自己的内在倾向。

### 页面结构

剧场分为四段：

1. 开场
2. 选择
3. 镜像
4. 尾声

### 第一幕：开场

目标：

建立场景，不解释规则。

页面只保留：

- 一句强场景文本
- 极弱展开入口
- 主 CTA

建议 CTA：

```text
进入这一幕
```

### 第二幕：选择

目标：

让用户靠近一个方向。

页面只保留：

- 当前场景句
- 单列选项卡

不显示：

- trait
- telemetry
- hover
- hesitation
- 内部记录
- 为什么要这样选

选项文案应像方向，而不是答案。

### 第三幕：镜像

目标：

让用户完成一次更内在的回答。

页面只保留：

- 一个问题
- 一句极短镜像提示
- 选项

镜像提示示例：

```text
你靠近的，也许不是答案，而是一种习惯。
```

### 尾声

目标：

收束剧场，引导进入星图。

页面只保留：

- 一句结束语
- 一句过渡
- 进入星图 CTA

CTA：

```text
生成我的星图
```

---

## 6.5 星图页 `/constellation`

### 页面目标

让用户在第一屏就获得强烈的「结果显形」感。

### 首屏结构

```text
原型中文名
英文副标题
一句本质摘要
分享 / 保存 / 重新探索
```

### 首屏要求

- 原型名必须是最大视觉焦点。
- 本质句必须短而击中。
- 操作区不能抢主视觉。
- 不显示技术信息。

### 默认正文结构

默认只展开三段：

1. 本质
2. 结构
3. 此刻

### 本质

只显示：

- 一条主线表达
- 一个次级支撑

不恢复四宫格摘要。

### 结构

只显示：

- 前三维
- 一个轻量切换
- 一段短解释

完整结构图折叠。

### 此刻

合并张力与路径。

只显示：

- 第一组张力
- 一条行动方向

### 折叠区

入口统一：

```text
继续阅读全部结果
```

折叠内容包含：

- 完整结构图
- 其余张力
- 其余路径
- 地形
- 回响
- 推荐内容

### 操作区

保留：

- 分享
- 保存
- 重新探索

视觉权重：

- 低于原型名
- 低于本质句
- 不做工具栏堆叠

---

## 7. 文案系统

### 7.1 总体语气

本源的文案应像镜子，不像老师。

它应该：

- 克制
- 诗性
- 深邃
- 不命令
- 不诊断
- 不鸡汤
- 不模板化

### 7.2 第二人称

使用「你」，不用「您」。

### 7.3 禁止风格

不使用：

- 系统正在处理
- 数据生成中
- 请完成以下问题
- 错误：字段缺失
- AI 正在分析
- 根据你的 trait signal
- 当前 provider

### 7.4 推荐替代表达

| 旧表达 | 新表达 |
|---|---|
| 请完成以下问题 | 先面对这一问 |
| 数据处理中 | 正在显影 |
| 上传图片文件 | 上传你的审美线索 |
| 测试结果 | 你的原型正在显形 |
| 发生错误 | 这一页暂时没有抵达 |
| 重新提交 | 再试一次 |

## 8. 视觉系统

### 8.1 视觉概念

从黑洞升空太空。

画面不是简单黑金，而是：

- 黑洞般的深场
- 深紫星云缓慢扩散
- 银白光晕照亮核心内容
- 金色只用于关键点亮
- UI 元素像漂浮在宇宙中的玻璃层

### 8.2 色彩

主色：

- 深黑
- 宇宙黑
- 暗紫
- 银白
- 冷白
- 暗金

慎用：

- 高饱和紫
- 塑料感蓝紫
- 大面积金色
- 红色 CTA

### 8.3 按钮

主按钮：

- pill 圆角
- 高度 56–60px
- 银白金渐变或暗金发光
- 文字清晰
- 按压有轻微下沉

次按钮：

- 玻璃 pill
- 低对比
- 不抢主按钮

### 8.4 卡片

卡片不应像后台面板。

应使用：

- 大圆角
- 半透明玻璃
- 细边框
- 柔和阴影
- 星云氛围背光

### 8.5 顶部

顶部只保留：

- 返回
- 极细进度
- 必要阶段感知

不保留：

- 网站导航
- 页面大标题栏
- lab 入口
- 多菜单

---

## 9. 交互原则

### 9.1 进度

用户只需要知道自己在哪里，不需要看到复杂流程图。

表现：

- 顶部 2px 细进度
- 极少文字
- 不显示完整模块面板

### 9.2 选择

选择应是沉浸行为，不是表单行为。

要求：

- 卡片面积足够大
- 触达区域清晰
- 选中状态明确
- 不显示内部解释

### 9.3 上传

上传应像交出素材，不像上传文件。

要求：

- 相册 / 拍照入口自然
- 上传后缩略图有美感
- 数量提示只一行

### 9.4 等待

等待不是中断，而是仪式。

要求：

- 不弹复杂 loading
- 不显示工程状态
- 动效慢、柔、深

### 9.5 结果阅读

结果不一次性砸给用户。

要求：

- 首屏先击中
- 默认三段短结果
- 完整内容折叠
- 用户愿意继续读才展开

---

## 10. 分享与保存

### 分享目标

让用户愿意把结果发给朋友。

分享内容应突出：

- 原型名
- 本质句
- 视觉氛围

不突出：

- 分数
- 图表
- 技术标签

### 保存目标

让用户把星图当成个人纪念物保存。

保存内容应像一张精神海报，而不是报告截图。

## 11. 当前产品验收标准

### 11.1 主流程验收

- 用户能从 `/collect` 顺畅进入 `/processing/benyuan`。
- 用户能从处理页进入 `/theater`。
- 用户能完成剧场选择。
- 用户能进入 `/constellation`。
- 全程不出现开发信息。

### 11.2 体验验收

- 每一屏只有一个主焦点。
- 用户不会感觉自己在填问卷。
- 用户不会看到技术解释。
- 页面切换有连续感。
- 等待状态有仪式感。
- 结果页第一屏有冲击力。

### 11.3 UI 验收

- 主流程视觉统一。
- 不再是旧黑金工作台感。
- 深紫不塑料。
- 银白 / 金色只做点亮。
- CTA 不使用突兀红色。
- iPhone 17 Pro Max 视口下自然成立。

### 11.4 文案验收

- 不鸡汤。
- 不诊断。
- 不报告腔。
- 不说明书腔。
- 有镜像感。
- 有节奏感。
- 短句优先。

1. `/collect`
2. `/processing/benyuan`

目标：

让这四页看起来像同一个原生 App，而不是四个网页页面。

用于保存和分享。

内容：

- 原型名

- 星云背景

默认只给：

- 首屏
- 本质
- 结构
- 此刻

观察：

- 用户是否愿意完成全部问题。

- 用户是否愿意分享结果。

## 13. 产品结论

本源目前已经不是「功能有没有」的问题，而是「体验是否足够本源」的问题。

当前产品的核心方向应从：

```text
心理测试 / 工作台 / 报告页
```

彻底转向：

```text
黑洞入口 / 精神剧场 / 星图显形
```

下一轮产品与设计工作的关键，是把所有多余信息撤掉，让用户在 iPhone 上只看到此刻最重要的一件事。

如果这件事成立，本源会从一个测试产品，变成一次值得完成、值得保存、值得分享的精神体验。
